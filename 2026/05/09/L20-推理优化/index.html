二、KV-Cache 详解

2.1 核心思想

KV-Cache 的思想非常简单：把已经计算过的 K 和 V 缓存起来，下次直接用。

每次前向传播时：

  • Q：只计算当前新 token 的 Query
  • K, V：只计算当前新 token 的 Key 和 Value，然后和缓存的历史 K/V 拼接

2.2 为什么只缓存 K 和 V，不缓存 Q？

在 Attention 计算中：

[
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V
]

  • Q 是当前 token 的查询，每次生成的 token 不同，Q 必须重新计算
  • K 是所有历史 token 的键，历史部分不会变，可以缓存
  • V 是所有历史 token 的值，同理可以缓存

  • Paged KV-Cache（vLLM 使用）：像操作系统的虚拟内存一样管理 KV-Cache
  • 量化 KV-Cache：用 INT8/INT4 存储 K/V

三、YaRN 长度外推

3.1 问题背景

MiniMind 训练时的最大序列长度（max_pos）是 32768。但如果用户输入了更长的文本怎么办？

朴素的 RoPE 不支持外推：如果位置编号超过训练时见过的范围，RoPE 的旋转角度会出现训练时没见过的模式，导致 Attention 分数紊乱，输出质量急剧下降。

3.2 YaRN 的基本思想

YaRN（Yet another RoPE extensioN）是一种 RoPE 的长度外推方法，核心思想是对旋转频率进行缩放，使得更长的序列也能映射到训练时见过的旋转角度范围。

传统 RoPE 的频率：

[
\theta_i = 10000^{-2i/d}
]

YaRN 对频率做缩放：

[
\theta_i’ = \frac{\theta_i}{s(i)}
]

其中 ( s(i) ) 是一个与维度位置 ( i ) 相关的缩放因子。YaRN 的关键贡献是将频率分为三个区域：

  1. 高频分量（变化快的维度）：不缩放（保持局部位置精度）
  2. 低频分量（变化慢的维度）：线性缩放（扩展长距离位置感知）
  3. 中间频率：平滑插值过渡

3.3 缩放效果

MiniMind 支持通过 YaRN 在推理时将上下文长度从 32768 扩展到：

[
32768 \times 16 = 524288 \approx 512K \text{ tokens}
]

这意味着推理时可以处理远超训练长度的输入，虽然效果可能不如在该长度上训练过的模型。

四、生成策略

4.1 贪心搜索（Greedy Search）

每一步选择概率最高的 token：

特点：确定性输出，速度最快，但容易陷入重复、缺乏多样性。

4.2 Temperature 参数

Temperature 控制输出概率分布的”锐利程度”：

[
p_i = \frac{\exp(z_i / T)}{\sum_j \exp(z_j / T)}
]

  • T < 1：分布更”尖锐”，高概率 token 更突出，输出更确定
  • T = 1：原始分布
  • T > 1：分布更”平坦”，低概率 token 有更多机会，输出更多样

直觉理解：Temperature 就像调节”创造力”的旋钮。T=0.1 写公文，T=1.0 聊天，T=1.5 写诗。

4.3 Top-K 采样

只从概率最高的 K 个 token 中采样：

问题：K 是固定的。有时候模型非常确定下一个 token（只有 2-3 个合理选择），K=50 会引入太多噪声；有时候模型不确定（10-20 个 token 都合理），K=50 可能仍然不够。

4.4 Top-P（Nucleus Sampling）

动态选择最小的 token 集合，使得累积概率达到 P：

优势：自适应——当模型确定时只选少数 token，不确定时选更多 token。

4.5 Top-K vs Top-P

维度 Top-K Top-P
候选数量 固定 K 个 动态，取决于概率分布
适应性 差（K 不变） 好（自适应分布形状）
参数 K（通常 50） P（通常 0.9-0.95）
适用场景 简单场景 更通用

实践中常组合使用：先 Top-K 过滤，再 Top-P 过滤，最后用 Temperature 调节。

4.6 Repetition Penalty（重复惩罚）

降低已经生成过的 token 的概率，防止”车轱辘话来回说”：

penalty > 1 时会降低已生成 token 的概率。典型值 1.1-1.3。

五、FlashAttention 简介

5.1 标准 Attention 的内存问题

标准 Attention 需要先计算完整的 ( QK^T ) 矩阵（大小 ( T \times T )），再做 Softmax，再乘 V。对于长序列，这个 ( T \times T ) 矩阵可能非常大。

5.2 FlashAttention 的核心思想

FlashAttention 通过**分块计算（tiling）**来避免存储完整的 Attention 矩阵：

  1. 将 Q、K、V 切成小块
  2. 在 GPU 的高速 SRAM 中计算小块的 Attention
  3. 用在线 Softmax 算法逐块累积结果
  4. 不需要在 HBM 中存储完整的 ( T \times T ) 矩阵

效果：

  • 内存从 ( O(T^2) ) 降到 ( O(T) )
  • 速度提升 2-4 倍（减少 HBM 读写）

六、模型量化简介

6.1 为什么需要量化

模型参数通常用 FP32（4 bytes）或 FP16（2 bytes）存储。量化就是用更少的位数表示：

6.2 量化类型

  • 权重量化（Weight Quantization）：只量化模型权重，推理时反量化回 FP16 计算
  • 激活量化（Activation Quantization）：同时量化权重和中间激活
  • KV-Cache 量化：量化 KV-Cache 以节省显存

6.3 常见量化方法

  • GPTQ：基于 Hessian 矩阵的逐层量化
  • AWQ：保护重要权重通道的量化
  • GGUF：llama.cpp 使用的量化格式，支持 Q4_K_M、Q5_K_M 等

6.4 量化与 MiniMind

MiniMind 支持转换为 GGUF 格式，使用 llama.cpp 推理时可以选择不同的量化级别。对于 64M 的小模型，量化的收益相对有限（模型本身就很小），但对于学习量化概念很有价值。

  2. 错。训练时整个序列并行输入，不需要 KV-Cache。
